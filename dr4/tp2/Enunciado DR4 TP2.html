Exercício 1 Lookup table byte→byte para normalização de código

Contexto

Em um projeto de engenharia da computação, é comum precisarmos normalizar códigos recebidos de periféricos ou protocolos internos antes de processá-los. Quando essa normalização ocorre milhares de vezes por segundo, cadeias de condicionais se tornam caras e imprevisíveis para o pipeline. Uma solução clássica em firmware é substituir decisões por uma lookup table, tornando o tempo de execução determinístico e reduzindo branches. Neste cenário, você recebeu uma tabela fixa de 256 entradas que mapeia um código de 8 bits para outro código de 8 bits, usada como camada de compatibilidade entre duas versões de um equipamento. Seu papel é implementar esse mapeamento em Assembly, garantindo que o acesso à memória seja correto e que a indexação não provoque leituras fora da tabela.

Tarefa

Implemente uma lookup table f com 256 bytes e uma rotina que receba um byte de entrada e retorne o byte de saída correspondente, usando o byte como índice direto da tabela.

Requisitos Técnicos e Saídas Esperadas

A tabela deve estar definida em memória (por exemplo, em .data) e a rotina deve calcular o endereço base + index. O professor validará executando o programa no Raspberry Pi e verificando, para um conjunto de entradas, se a saída corresponde aos bytes definidos na tabela.

Exercício 2 Lookup table byte→halfword com indexação por elemento

Contexto

Tabelas de consulta não retornam apenas bytes: é comum mapear entradas pequenas para coeficientes, fatores de calibração ou parâmetros prontos para uso, que podem ocupar 16 bits. Nesses casos, um erro típico é tratar a tabela como se fosse de bytes e acessar o endereço errado, retornando valores “quase aleatórios”. Neste cenário, a equipe de hardware definiu uma tabela de 256 valores de 16 bits, onde cada índice representa um nível de sinal e cada saída é o valor já escalado. Você deve implementar o acesso correto em Assembly ARM64, respeitando o tamanho do elemento e garantindo que o offset seja multiplicado por 2.

Tarefa

Implemente uma lookup table f com 256 halfwords (16 bits) e uma rotina que receba um byte de entrada e retorne o halfword de saída correspondente.

Requisitos Técnicos e Saídas Esperadas

A rotina deve calcular o endereço base + (index * 2) e carregar um halfword. O professor validará a execução no Raspberry Pi comparando saídas para ao menos 4 entradas distintas.

Exercício 3 Lookup table word→byte com derivação explícita de índice

Contexto

Em sistemas reais, a entrada para uma tabela pode ser maior do que o índice permitido. Por exemplo, um sensor pode produzir uma word com flags e campos compactados, mas apenas um subconjunto de bits deve ser usado como índice. Se essa derivação não for explícita, dois engenheiros podem implementar regras diferentes e o sistema se torna inconsistente. Neste cenário, você recebe uma word de entrada, derivará um índice de 8 bits de forma explícita e consultará uma tabela de bytes. O objetivo é avaliar se você controla a transição entre “valor amplo” e “índice” de maneira disciplinada.

Tarefa

Implemente uma rotina que receba um word (32 bits), derive um índice de 8 bits (por exemplo, usando os 8 bits menos significativos) e retorne um byte obtido de uma lookup table.

Requisitos Técnicos e Saídas Esperadas

No relatório, descreva claramente como o índice é derivado do word. O professor validará com entradas distintas que produzem índices diferentes e acessam posições diferentes da tabela.

Exercício 4 Lookup table word→word com indexação por 4 bytes

Contexto

Em tabelas de parâmetros, é comum que a saída seja um valor de 32 bits pronto para ser escrito em outro registrador ou usado em cálculos. Isso exige indexação por elementos de 4 bytes. Um erro frequente é esquecer o escalonamento do offset e acessar uma word “no meio” de outra, corrompendo a leitura. Neste cenário, você implementará uma tabela de words e acessará o elemento correto a partir de um índice, com offset multiplicado por 4.

Tarefa

Implemente uma lookup table com elementos de 32 bits e uma rotina que retorne o word correto, calculando o endereço base + (index * 4).

Exercício 5 Limpeza de bits por máscara usando BIC

Contexto

Em controle de hardware e estruturas compactadas, limpar bits é uma operação constante: desligar recursos, limpar flags de erro, resetar campos antes de inserir novos valores. Em ARM64, isso deve ser feito de forma direta e sem ramificações, aplicando uma máscara que zere apenas os bits desejados. Neste cenário, você precisa preparar um valor base para receber novos campos, limpando previamente a região controlada por uma máscara.

Tarefa

Implemente uma rotina que receba um valor base e uma máscara e produza um valor com os bits indicados pela máscara zerados, preservando os demais.

Exercício 6 Ativação de bits por máscara usando ORR

Contexto

Habilitar bits específicos sem alterar campos adjacentes é essencial para configuração segura de modos e recursos. Em firmware, a operação deve ser idempotente e previsível: aplicar a máscara sempre produz o mesmo efeito. Neste exercício, você implementará a habilitação de flags e documentará o comportamento para casos em que alguns bits já estejam setados.

Tarefa

Implemente uma rotina que receba um valor base e uma máscara e retorne o valor com os bits da máscara forçados para 1.

Exercício 7 Alternância de bits por máscara usando EOR

Contexto

Em máquinas de estado e sinalização, alternar bits é mais eficiente do que testar e decidir via branches. A operação XOR permite alternância determinística com uma máscara, mantendo previsibilidade do pipeline. Neste cenário, você foi instruído a implementar um “toggle” de flags com base em uma máscara, garantindo que apenas os bits selecionados invertam.

Tarefa

Implemente uma rotina que receba um valor e uma máscara e retorne o valor com apenas os bits da máscara invertidos.

Exercício 8 Teste de bits por mascaramento e retorno de status

Contexto

Muitos fluxos de firmware precisam responder rapidamente à pergunta: “há algum erro ativo?”, “algum modo está habilitado?”, “existe qualquer bit setado nesse grupo?”. Em vez de múltiplas comparações, o padrão é mascarar e testar zero/não-zero, retornando um status simples para o chamador. Este exercício avalia sua capacidade de construir essa verificação e padronizar uma saída objetiva.

Tarefa

Implemente uma rotina que receba um valor e uma máscara e retorne:

  • 1 se pelo menos um bit do grupo estiver setado,
  • 0 se todos os bits do grupo estiverem zerados.

Exercício 9 Extração de campo por máscara e shift (alinhamento à direita)

Contexto

Campos empacotados aparecem em protocolos, registradores e formatos binários. Para interpretar um campo, é necessário isolar a região e alinhar o resultado, normalmente deslocando o campo para o bit menos significativo. Um erro de máscara ou shift gera valores coerentes apenas por coincidência em alguns casos. Neste exercício, você extrairá um campo com largura e offset definidos explicitamente, retornando o valor alinhado à direita.

Tarefa

Implemente uma rotina que extraia de um valor de 64 bits um campo definido por offset e largura, retornando o campo alinhado à direita (LSB).

Exercício 10 Inserção de campo: limpar região + posicionar + ORR

Contexto

Inserir campos em uma palavra existente é um padrão de empacotamento. A técnica correta envolve limpar primeiro o campo no valor base, posicionar o novo valor com shift e combinar com ORR. Soluções que apenas “ORR direto” podem sobrescrever bits antigos de forma incorreta. Neste cenário, você implementará uma inserção completa e verificável de campo, preservando o restante da palavra.

Tarefa

Implemente uma rotina que receba um valor base de 64 bits e um valor de campo, e atualize o campo definido por offset e largura, preservando os demais bits.

Exercício 11 Rotação manual à esquerda (rotate left) sem instrução dedicada

Contexto

Rotações são usadas em funções de hash, checksums e transformações de dados em baixo nível. Diferente de shift, rotação não descarta bits: os bits que saem por um lado retornam pelo outro. Implementar rotação manual é um exercício fundamental para consolidar o controle sobre largura de palavra e comportamento de bits. Neste cenário, você implementará rotate-left como composição de dois shifts e OR, garantindo preservação completa dos bits.

Tarefa

Implemente uma rotina que realize rotação à esquerda de um valor de 64 bits por N posições, usando apenas shifts e OR.

Exercício 12 Pipeline completo: extração → lookup → ajuste final

Contexto

Em rotinas de otimização escalar, é comum combinar técnicas: um valor chega compactado, um campo vira índice para uma tabela, a saída da tabela ainda precisa ser reposicionada e combinada com outros bits. Esse tipo de pipeline substitui decisões por transformações determinísticas e permite desempenho alto com previsibilidade. Neste exercício final, você irá montar uma rotina completa que aplica três etapas encadeadas, com clareza, testes e justificativa técnica. A correção valorizará tanto o resultado quanto a auditabilidade do pipeline.

Tarefa

Implemente uma rotina que:

  1. Extraia um campo de bits de um valor de entrada de 64 bits (offset e largura definidos no seu código);
  2. Use esse campo como índice em uma lookup table;
  3. Ajuste o valor retornado aplicando máscara e posicionamento por shifts e ORR.

Requisitos Técnicos e Saídas Esperadas

No relatório, descreva cada etapa e apresente pelo menos 3 casos de teste. O professor validará a coerência entre o campo extraído, o índice usado, o elemento lido na tabela e o valor final produzido.

    Assessment [Obrigatório]

    Olá, Renato.

    Chegamos ao momento de avaliar o conhecimento construído ao longo da disciplina DR1 – Fundamentos de Verilog e FPGA.

    Uso de IAs: Sinal Vermelho 🔴

    Todas as partes deste trabalho devem ser da autoria do aluno. Qualquer uso de ferramentas generativas de IA, como ChatGPT, é proibido. O uso de IA generativa será considerado má conduta acadêmica e estará sujeito à aplicação do código disciplinar, pois as tarefas deste trabalho foram elaboradas para desafiar o aluno a desenvolver conhecimentos de base, pensamento crítico e habilidades de resolução de problemas. O uso da tecnologia de IA limitaria sua capacidade de desenvolver essas competências e de atingir os objetivos de aprendizagem desta disciplina.

    Ao longo do curso você aprendeu a representar sistemas digitais por meio de lógica booleana, implementar circuitos combinacionais e sequenciais simples em Verilog HDL, criar testbenches para validação funcional, interpretar formas de onda de simulação e executar projetos digitais em FPGA real utilizando a Tang Nano 4K.

    Neste Assessment Final você irá aplicar esses conhecimentos no desenvolvimento de um sistema digital completo, passando por todas as etapas do fluxo de engenharia digital:

    • especificação funcional do sistema
    • modelagem lógica usando álgebra booleana
    • implementação em Verilog HDL
    • criação de testbench para simulação
    • análise de waveforms
    • síntese e execução em FPGA real (Tang Nano 4K)

    O projeto deve ser tratado como um mini-projeto profissional de engenharia digital, com organização adequada, clareza técnica e validação completa do funcionamento.

    Projeto – Sistema Digital de Controle de Acesso para Porta Eletrônica

    Portas eletrônicas simples são amplamente utilizadas em ambientes como laboratórios, salas técnicas e equipamentos industriais. Esses sistemas verificam sinais digitais provenientes de sensores ou teclados e determinam se o acesso deve ser liberado.

    Neste projeto você deverá desenvolver um Sistema Digital de Controle de Acesso implementado em FPGA, capaz de verificar um código digital de entrada, registrar tentativas de acesso e sinalizar o resultado da verificação.

    O sistema será implementado utilizando Verilog HDL e executado na Tang Nano 4K, utilizando entradas físicas da placa e indicadores visuais através dos LEDs.

    Interface do Sistema

    O sistema deverá utilizar as seguintes entradas e saídas da Tang Nano 4K.

    Entradas

    Sinal Descrição
    SW0 Bit 0 do código de acesso
    SW1 Bit 1 do código de acesso
    SW2 Bit 2 do código de acesso
    SW3 Bit 3 do código de acesso
    BTN1 Botão de confirmação de acesso
    BTN2 Reset do sistema

    Os switches representam o código digitado pelo usuário.
    O botão BTN1 indica que o usuário deseja validar o código digitado.
    O botão BTN2 reinicia o sistema.

    Saídas

    LED Função
    LED0 Acesso autorizado
    LED1 Acesso negado
    LED2 Tentativa registrada
    LED3 Bit 0 do contador de tentativas
    LED4 Bit 1 do contador de tentativas
    LED5 Bit 2 do contador de tentativas

    Código de Acesso

    O sistema deverá verificar se o código presente nos switches corresponde ao seguinte código válido: 1011

    SW3 SW2 SW1 SW0
    1 0 1 1

    A verificação do código deve ser implementada usando lógica combinacional baseada em expressões booleanas.

    Modelagem Lógica Obrigatória

    Antes da implementação em Verilog, o aluno deverá modelar formalmente a lógica de verificação do código.

    Essa modelagem deve incluir:

    • tabela verdade da verificação do código
    • expressão booleana correspondente
    • simplificação da expressão usando álgebra booleana ou mapas de Karnaugh

    A implementação em Verilog deve corresponder à expressão lógica obtida.

    Requisitos Funcionais

    Verificação do código

    Quando o botão BTN1 for pressionado:

    1. O sistema deve verificar o valor presente nos switches SW0–SW3.
    2. O valor deve ser comparado com o código válido 1011.

    Se o código estiver correto:

    • LED0 deve acender
    • LED1 deve permanecer apagado

    Se o código estiver incorreto:

    • LED1 deve acender
    • LED0 deve permanecer apagado

    Registro de tentativa

    Sempre que BTN1 for pressionado:

    • o sistema deve registrar uma tentativa
    • LED2 deve indicar que ocorreu uma tentativa

    Contador de tentativas

    O sistema deve possuir um contador síncrono de 3 bits, exibido nos LEDs:

    LED Significado
    LED3 Bit 0
    LED4 Bit 1
    LED5 Bit 2

    O contador deve:

    • incrementar a cada tentativa de acesso
    • ser implementado usando registradores ou flip-flops em Verilog
    • operar sincronizado com o clock da FPGA

    Reset do sistema

    Quando BTN2 for pressionado:

    • o contador deve ser zerado
    • todos os LEDs de status devem ser apagados

    Requisitos de Implementação

    O projeto deve ser implementado em Verilog HDL e conter:

    • módulos com portas de entrada e saída bem definidas
    • lógica combinacional para verificação do código
    • circuito sequencial para o contador
    • separação clara entre lógica combinacional e lógica sequencial

    Simulação

    Antes da implementação em hardware, o projeto deve ser validado por simulação.

    O aluno deverá criar um testbench dedicado contendo:

    • geração de clock
    • aplicação de reset
    • cenários de teste para:
      1. código correto
      2. código incorreto
      3. múltiplas tentativas
      4. reset do sistema

    As formas de onda geradas na simulação devem demonstrar:

    • verificação do código
    • funcionamento do contador
    • comportamento das saídas

    Implementação em FPGA

    Após validação por simulação, o projeto deve ser:

    1. sintetizado
    2. mapeado para a FPGA
    3. programado na Tang Nano 4K

    O funcionamento deve ser demonstrado utilizando:

    • switches e botões como entradas
    • LEDs como indicadores de saída

    Formato de Entrega

    A entrega do Assessment Final deve seguir rigorosamente os critérios abaixo.

    • Todos os arquivos devem ser compactados em um único arquivo .ZIP.

    O arquivo deve conter:

    1. Código do projeto

    • arquivos Verilog (.v)
    • arquivo de constraints
    • arquivos de testbench

    2. Relatório técnico

    • descrição do sistema desenvolvido
    • arquitetura do circuito
    • tabela verdade e expressão booleana da verificação do código
    • decisões de projeto adotadas
    • interpretação das formas de onda da simulação

    3. Evidências de funcionamento

    • imagens ou vídeos demonstrando o funcionamento do circuito na Tang Nano 4K

    Vídeo demonstrativo

    O vídeo deve mostrar:

    • funcionamento do sistema
    • interação com switches e botões
    • resposta observada nos LEDs

    O vídeo deve:

    • estar disponível como link do YouTube ou Drive
    • estar configurado como não listado
    • possuir permissão de visualização habilitada

    Orientações para Defesa do Trabalho em Vídeo

    1. Utilize uma ferramenta adequada: É preciso que apareça a tela com seu trabalho e a imagem da sua webcam. Uma sugestão de ferramenta gratuita online para estudantes é o Loom. Caso prefira, existe também a opção de baixar a ferramenta OBS, dentre outras.
    2. Escolha um local adequado: Grave em um ambiente calmo, sem ruídos ou distrações, para garantir boa qualidade de áudio e foco durante a defesa.
    3. Webcam ligada: Certifique-se de que a webcam esteja ativada, com boa iluminação e enquadrando você de forma clara e centralizada.
    4. Revise o vídeo: Assista a gravação para verificar se o vídeo e o áudio estão bons.
    5. Salve o vídeo no Google Drive: Após a gravação, faça o upload do arquivo para o seu Google Drive de aluno Infnet.
    6. Altere as permissões de acesso: Configure o compartilhamento do vídeo para "qualquer pessoa com o link" poder visualizar. Copie o link gerado e envie conforme as instruções da atividade.

    Assim que terminar, salve seu trabalho em PDF nomeando o arquivo conforme a regra “nome_sobrenome_DR1_AT.PDF” e poste como resposta a este AT.
